The implementation within this repository is organized into five tightly bound layers:
1. **Layer 1 (Visual Control)**: `index.html` featuring the real-time SSPA Persistent State Identity Sync Indicator.
2. **Layer 2 (Automation)**: `banking_bridge.py` cryptographic routing pipeline with local Node.js Webhook Server on port `8082`.
3. **Layer 3 (Persistence)**: SSPA Core Identity Vault validating user status using the strict sequence: `PIN ──► PID ──► PIH`.
4. **Layer 5 (Contracts & Node)**: Deployed under Solidity v0.8.20 (`QSTATE.sol` & `SovereignBank.sol`) linked with Chainstack Elastic Cloud nodes (`jolly-goldberg`).
